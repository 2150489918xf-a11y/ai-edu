# 教务端标准后台管理系统改造设计

## 改造目标

将当前把教师表单、教师列表、学生列表和课程分配同时堆叠在一个页面中的教务端，改造成独立、清晰、可扩展的标准后台管理系统，并补齐教师、学生、班级、课程的增删改查与恢复能力。

改造必须保持教师端、学生端和教务端相互独立，不在教师端或学生端导航中暴露教务入口。

## 现状问题

- 所有管理任务集中在 `AdminOfficePage.vue`，页面同时承担布局、筛选、表单、列表、选中态和课程分配。
- 新增教师表单长期占据内容区，列表和操作层级不清晰。
- 学生列表与课程分配强耦合，班级和课程没有独立管理入口。
- 页面使用渐变、玻璃卡片和大标题，更像展示型页面而不是高频操作后台。
- 教师只有新增、编辑和停用，缺少恢复入口；学生、班级、课程缺少教务端完整 CRUD。

## 信息架构

教务端采用独立后台框架和子路由：

| 路由 | 页面 | 主要功能 |
| --- | --- | --- |
| `/admin` | 自动跳转 | 跳转至 `/admin/overview` |
| `/admin/overview` | 工作台 | 核心统计、管理快捷入口 |
| `/admin/teachers` | 教师管理 | 查询、新增、编辑、停用、恢复 |
| `/admin/students` | 学生管理 | 查询、新增、编辑、调整班级、停用、恢复 |
| `/admin/classes` | 班级管理 | 查询、新增、编辑、指定教师、停用、恢复 |
| `/admin/courses` | 课程管理 | 查询、新增、编辑、指定教师、停用、恢复 |
| `/admin/enrollments` | 课程分配 | 查看班级课程，新增或移除个人课程 |

`/admin/login` 保持独立登录页。所有 `/admin/**` 路由继续要求 `admin` 角色。

## 页面框架

新增 `AdminLayout.vue` 作为教务端唯一框架：

- 左侧固定导航，宽屏为完整菜单，平板折叠，移动端变为抽屉；
- 顶部栏显示菜单按钮、面包屑、刷新入口、管理员身份和退出登录；
- 内容区只渲染当前模块，不再一次加载全部管理数据；
- 页面背景使用冷灰色，导航使用深灰绿色，内容使用白色表格面板；
- 取消装饰性渐变、玻璃拟态和衬线大标题；
- 表格行、筛选栏、按钮和状态标签遵循紧凑后台密度。

新增和编辑使用统一右侧抽屉；停用使用确认对话框；加载、空数据、错误和成功反馈均在当前页面内展示。

## 管理模块

### 工作台

展示教师、学生、班级、课程和个人课程分配数量。每项统计卡可进入对应模块，并提供“新增教师”“新增学生”“课程分配”等快捷操作。

### 教师管理

- 支持姓名、账号、手机号和邮箱关键字查询；
- 支持有效、已停用和全部状态筛选；
- 新增时创建教师账号和教师档案；
- 编辑姓名、电话和邮箱；
- 停用后同时停用登录账号；
- 恢复后同时恢复登录账号。

### 学生管理

- 支持姓名、学号和账号关键字查询；
- 支持班级与状态筛选；
- 新增时创建学生账号并指定班级；
- 编辑姓名、学号和班级；
- 停用后保留答题、选课、画像与家长摘要数据，同时停用登录账号；
- 恢复后恢复登录账号。

### 班级管理

- 支持班级名称、年级、学科和状态查询；
- 新增与编辑班级名称、年级、学科和负责人；
- 有有效学生或有效课堂的班级不允许停用，必须先迁移学生并处理课堂；
- 空班级可停用并恢复。

### 课程管理

- 支持课程名称、学科、年级、教师和状态查询；
- 新增与编辑课程基础信息和任课教师；
- 停用课程只修改状态，不删除课件、PPT、思维导图、教案、题目、答题和分析；
- 已停用课程不能新增课程分配，但可恢复。

### 课程分配

- 主区为可搜索、可按班级筛选的学生表格；
- 点击学生后打开右侧详情抽屉；
- 抽屉分为班级课程、个人课程和可分配课程；
- 班级课程只读，个人课程可移除，可分配课程可添加；
- 已停用学生或课程不允许创建新分配。

## API 设计

保留 `/api/v1/admin/summary` 与现有课程分配接口，补齐以下资源接口：

```text
GET    /api/v1/admin/teachers
POST   /api/v1/admin/teachers
PATCH  /api/v1/admin/teachers/:id
DELETE /api/v1/admin/teachers/:id
POST   /api/v1/admin/teachers/:id/restore

GET    /api/v1/admin/students
POST   /api/v1/admin/students
PATCH  /api/v1/admin/students/:id
DELETE /api/v1/admin/students/:id
POST   /api/v1/admin/students/:id/restore

GET    /api/v1/admin/classes
POST   /api/v1/admin/classes
PATCH  /api/v1/admin/classes/:id
DELETE /api/v1/admin/classes/:id
POST   /api/v1/admin/classes/:id/restore

GET    /api/v1/admin/courses
POST   /api/v1/admin/courses
PATCH  /api/v1/admin/courses/:id
DELETE /api/v1/admin/courses/:id
POST   /api/v1/admin/courses/:id/restore
```

列表接口统一返回 `data` 和 `pagination`，支持 `page`、`pageSize`、`keyword`、`status` 及资源特有筛选字段。

## 数据安全规则

- 所有“删除”均为软删除：资源状态改为 `archived` 并写入 `deletedAt`；
- 恢复时状态改回 `active` 并清除 `deletedAt`；
- 教师和学生停用、恢复时同步更新关联 `User.status`；
- 不级联物理删除历史业务数据；
- 班级停用前检查有效学生和课堂；
- 课程停用后禁止创建新选课，但历史记录保持可查询；
- 前端危险操作必须二次确认，并展示后端返回的具体阻止原因。

## 组件边界

- `AdminLayout.vue`：导航、顶栏、移动菜单和退出登录；
- `AdminOverviewPage.vue`：只管理工作台统计；
- 四个资源管理页：分别管理筛选、表格、分页和资源表单；
- `AdminEnrollmentsPage.vue`：只管理学生课程分配；
- `AdminResourceDrawer.vue`：统一侧边抽屉容器；
- `AdminConfirmDialog.vue`：统一危险操作确认；
- `AdminStatusBadge.vue`：统一状态展示；
- `adminApiClient.js`：统一教务资源 API 与缓存失效。

后端继续由 `adminService.js` 负责教务业务规则，HTTP 路由仅做参数解析和响应转换。

## 响应式设计

- `>= 1024px`：固定侧栏、完整表格、右侧抽屉；
- `768px–1023px`：折叠侧栏，内容区保持表格并允许横向滚动；
- `< 768px`：侧栏变为覆盖式抽屉，筛选项纵向排列，编辑抽屉占满屏幕；
- 所有交互控件最小触控区域为 44×44 像素；
- 表格在窄屏保持列结构并提供横向滚动，不将每行强行拆成杂乱卡片。

## 验证标准

- 教务管理员可通过独立路由进入六个模块；
- 教师、学生、班级、课程的新增、查询、编辑、停用和恢复均可真实写入数据库；
- 非空班级停用会得到明确错误且数据库不改变；
- 停用课程不会删除关联教学内容与学习数据；
- 课程分配原有功能保持可用；
- 教师端和学生端不出现教务导航；
- 桌面 1440px、笔记本 1024px、平板 768px、手机 375px 均无页面级横向溢出或遮挡；
- 现有认证、教务 API、缓存和生产构建测试通过。
