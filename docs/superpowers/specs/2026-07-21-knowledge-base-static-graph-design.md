# 知识库静态知识图谱设计

## 目标

在教师端“知识库管理”页面增加知识图谱入口，提供一个与题库知识路径图视觉一致的只读示例页面。图谱数据完全由前端静态文件提供，不访问数据库、AI 服务或新增后端接口。

## 已确认范围

- 入口位于知识库管理页右上角，与“批量导入”“上传资料”处于同一操作区。
- 新页面路由为 `/knowledge-base/knowledge-graph`。
- 使用现有四阶段教学路径：基础概念、核心规律、解题方法、综合应用。
- 默认仅展示相邻阶段主路径；点击节点后展示与该节点相关的隐藏关系。
- 保留搜索、主题筛选、重新分层、适应画布、缩放和节点详情。
- 图谱为只读模式，不提供新增、编辑、删除、合并、解除关联、AI 解析或数据协调操作。
- 不修改数据库 Schema、种子数据、服务端路由和现有题库图谱数据流。

## 方案选择

采用“独立页面 + 复用渲染器 + 独立静态数据模块”。

未采用的方案：

1. 直接复用 `QuestionBankKnowledgeGraphPage.vue` 并增加模式分支。该页面包含轮询、AI 解析和完整 CRUD，静态知识库图谱会被无关逻辑耦合。
2. 复制题库图谱页面和渲染代码。开发较快，但会形成两套视觉与交互实现，后续样式调整容易不一致。

选择方案将题库和知识库的数据来源保持独立，同时共享稳定的路径投影与 G6 渲染能力。

## 页面与组件设计

### 知识库入口

在 `KnowledgeBasePage.vue` 的 `.hero-actions` 中加入“知识图谱”软按钮，图标使用 `account_tree`，点击后通过 Vue Router 进入 `/knowledge-base/knowledge-graph`。

### 知识库图谱页面

新增 `KnowledgeBaseKnowledgeGraphPage.vue`，沿用题库图谱页面的头部、状态条、工具栏、画布卡片和右侧详情栏布局。

页面结构：

- 头部：返回知识库、页面标题、静态示例说明。
- 状态条：资料数量、知识点数量、路径关系数量、四个教学阶段。
- 工具栏：知识点搜索、主题筛选、重新分层、适应画布。
- 主区域：左侧 `KnowledgeGraphRenderer`，右侧只读节点详情。
- 未选择节点时：右侧显示操作引导。

窄屏沿用现有知识路径图规则：画布与详情上下排列，由页面承担纵向滚动，不产生横向溢出。

### 可复用边界

- 继续复用 `KnowledgeGraphRenderer.vue` 负责 G6 绘制、节点选中和关系显隐；为渲染器增加可选计数单位与图例文案，默认值仍为题库的“道题”，知识库页面传入“条资料”。
- 继续复用 `teachingPathProjection.js` 负责四阶段归并和固定坐标。
- 新增轻量只读详情组件，避免在静态页面加载 `KnowledgeGraphInspector.vue` 的保存、合并和删除功能。
- 不改变现有渲染器的输入协议：`graphData` 仍为 `{ nodes, edges }`。

## 静态数据设计

新增 `src/data/knowledgeBaseGraphMock.js`，导出知识库概要和图谱数据。数据聚焦高中物理力学知识体系，使演示关系完整且容易理解。

节点字段：

```js
{
  id: 'force-concept',
  label: '力的基本概念',
  category: '力学基础',
  questionCount: 8,
  source: 'explicit',
  description: '描述力的作用效果、三要素及常见表示方式。',
  aliases: ['力的三要素', '力的表示'],
  materialRefs: ['高中物理必修一', '力学基础讲义']
}
```

关系字段：

```js
{
  id: 'force-to-newton-two',
  source: 'force-concept',
  target: 'newton-second-law',
  type: 'prerequisite',
  label: '前置知识',
  supportCount: 3
}
```

示例数据包含 16 至 20 个节点，并覆盖以下路径：

- 基础概念：力、质量、速度、加速度、参考系。
- 核心规律：牛顿运动定律、动量定理、机械能守恒。
- 解题方法：受力分析、正交分解、整体隔离法、临界条件分析。
- 综合应用：斜面系统、连接体、传送带、碰撞与综合动力学。

静态节点保留 `questionCount` 作为“关联资料证据数”的视觉权重。知识库页面通过渲染器可选属性把节点计数显示为“条资料”，图例显示为“关联资料越多，节点证据越充分”，不宣称这些数据来自真实解析。

## 数据流

1. 页面同步导入静态图谱数据。
2. 搜索与主题筛选在前端计算可见节点，并过滤无效关系。
3. `KnowledgeGraphRenderer` 调用四阶段投影生成固定坐标和默认可见关系。
4. 用户选择节点后，页面记录节点 ID；渲染器展开关联隐藏路径，只读详情组件显示节点说明、别名、引用资料和上下游关系。
5. 刷新页面后重新读取相同静态数据，不产生持久化写入。

## 错误与空状态

- 静态数据无需加载状态或网络错误提示。
- 搜索或主题筛选无结果时显示“没有匹配的知识点”，并提示清除筛选。
- 未选择节点时右侧显示点击节点的引导。
- 静态数据中无效端点关系由现有路径投影自动过滤，页面不崩溃。

## 测试与验收

自动测试覆盖：

- 路由存在且使用独立页面。
- 知识库管理页包含知识图谱入口和正确跳转地址。
- 静态数据具有四阶段可投影路径、稳定 ID、有效关系端点和足够节点数量。
- 页面复用 `KnowledgeGraphRenderer`，不包含题库 CRUD、轮询或 API 调用。
- 题库继续显示“道题”计数，知识库显示“条资料”计数。
- 只读详情包含说明、别名、引用资料和上下游关系。
- 原有题库知识图谱测试保持通过。

浏览器验收覆盖：

- 从知识库管理页可进入图谱并返回。
- 桌面端显示四阶段背景、主路径和右侧详情。
- 点击节点后详情更新并展开相关隐藏路径。
- 搜索与主题筛选正常。
- 768px 和 375px 下无横向溢出，页面可纵向滚动。
- 控制台无错误。

## 非目标

- 不根据知识库文件自动抽取知识点。
- 不实现知识库图谱 CRUD。
- 不把静态数据写入 PostgreSQL。
- 不接入 AI 生成、增量更新、版本管理或人工编辑。
- 不改变题库知识图谱页面的功能和数据。
