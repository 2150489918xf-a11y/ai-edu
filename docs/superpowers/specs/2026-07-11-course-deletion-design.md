# 课程分组与备课单元删除设计

## 目标

教师可以删除空课程分组，也可以永久删除一个备课单元。删除备课单元时，系统必须同步清理该单元生成的课件数据、题目、答题记录和学习分析数据，不能留下孤立数据。

## 删除边界

### 课程分组

- 只有 `unitCount = 0` 的课程分组允许删除。
- 后端必须再次统计分组下所有状态的备课单元，不能只信任前端显示的数量。
- 非空分组返回 HTTP 409，错误码为 `COURSE_GROUP_NOT_EMPTY`，并返回 `unitCount`。
- 空分组删除时，其学生课程分组分配记录由数据库级联清理。

### 备课单元

- 现有“归档/恢复”功能保留。
- 新增独立的“永久删除”入口，避免改变现有归档接口语义。
- 永久删除在一个 Prisma 事务内完成。
- 显式删除 `courseId` 对应的题目，因为当前题目外键使用 `onDelete: SetNull`。
- 删除题目后，题目知识点关系、课堂题目关系和学生答题记录通过外键级联清理。
- 删除备课单元后，知识点、课堂任务、学生单元分配、学习画像和家长分析通过外键级联清理。
- 大纲、思维导图、教案和 PPT 属于备课单元记录，随记录一起删除。
- 清理知识点前先解除同课程知识点的父子关系，避免自关联约束阻止删除。

## API

- `DELETE /api/v1/course-groups/:groupId`：仅删除空分组。
- `DELETE /api/v1/courses/:courseId/permanent`：永久删除备课单元及其关联数据。
- `DELETE /api/v1/courses/:courseId`：继续表示归档。

成功响应包含删除对象 id 和清理摘要；不存在返回 404；非空分组返回 409。

## 教师端交互

- 每个课程分组筛选项旁显示删除按钮。
- 非空分组的删除按钮禁用，并提示“请先删除分组下的备课单元”。
- 空分组删除前显示确认提示。
- 备课单元详情操作区新增红色“永久删除”按钮，确认文案明确列出课件、题目、作答和分析会一起删除。
- 删除成功后重新加载课程与分组，并选择当前筛选范围内的第一个备课单元。

## 数据模型补充

在 `Course` 增加可空的 `ppt Json?` 字段，与现有 `outline`、`mindmap`、`lessonPlan` 保持相同的课程级持久化方式，为真实 PPT 生成结果提供删除边界。

## 验证

- API 合同测试覆盖两个新删除路由及 409 错误。
- 数据库测试创建完整关联数据，验证备课单元删除后题目、答题、任务、画像和课程记录均不存在。
- 数据库测试验证空分组可删除、非空分组被拒绝。
- 前端 API 客户端和页面合同测试覆盖删除函数、确认提示和按钮状态。
- 运行现有课程测试及生产构建，防止破坏归档、恢复和备课工作台。
