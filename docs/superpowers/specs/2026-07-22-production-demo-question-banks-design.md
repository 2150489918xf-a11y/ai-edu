# 生产环境多学科演示题库数据设计

## 目标

在不清空、不覆盖用户业务数据的前提下，为生产环境补充可展示、可检索、可进入详情并可打开知识图谱的多学科题库数据。

完成后至少包含：

- 15 个新增演示题库，连同现有牛顿定律题库共 16 个；
- 物理、数学、化学、英语、语文 5 个学科；
- 每个新增题库 12 道选择题或填空题，共 180 道新增题目；
- 每个题库 6 个知识点、知识点与题目的关联、稳定的前置关系；
- 题库题目尽可能关联现有课程，使教师端题库和课程数据保持一致。

## 方案比较

### 方案一：只创建题库外壳

改动最小，但题库详情、统计与知识图谱为空，不能形成有效演示。

### 方案二：运行时随机生成大量记录

数量增长快，但重复执行会产生重复数据，内容不可预测，也不适合生产环境回滚和验收。

### 方案三：固定教学目录并幂等灌入（采用）

将演示目录作为代码内的确定性数据集，使用固定 ID 和 Prisma `upsert` 写入。重复执行只会校正演示记录，不会重复创建，也不会删除用户记录。

## 数据结构

新增两个职责分离的模块：

1. `prisma/demoQuestionBankCatalog.js`：只描述题库、知识点、关系和题目，不连接数据库。
2. `prisma/seedDemoQuestionBanks.js`：读取目录，通过事务与 `upsert` 写入数据库。

题库覆盖以下主题：

- 物理：匀变速直线运动、力的合成与平衡、机械能、动量；
- 数学：函数基础、二次函数、三角函数、平面向量；
- 化学：物质的量、氧化还原反应、离子反应；
- 英语：定语从句、阅读理解；
- 语文：现代文阅读、文言文基础。

每个题库包含 6 个知识点，按“基础概念 → 核心规律 → 方法策略 → 综合应用”组织。12 道题同时包含单项选择题和填空题，题目通过 `QuestionKnowledgePoint` 关联 1 至 2 个知识点。

## 幂等与安全

- 所有新增 ID 使用 `demo-bank-*`、`demo-kp-*`、`demo-q-*` 前缀；
- `QuestionBank`、`KnowledgePoint`、`Question`、`QuestionKnowledgePoint` 和 `KnowledgeRelation` 均使用稳定键 `upsert`；
- 不调用 `deleteMany`，不归档、不修改非 `demo-*` 数据；
- 题目只关联数据库中真实存在的课程；课程不存在时允许 `courseId = null`，不会导致脚本失败；
- 运行前对生产 PostgreSQL 创建备份，运行后查询数量并抽查 API。

## 验证

自动测试验证目录规模、ID 唯一性、学科覆盖、题型覆盖、答案完整性、知识点引用合法性和禁止删除逻辑。部署验证包括：

- 脚本连续执行两次后记录数量不增长；
- 题库列表 API 返回至少 16 个有效题库；
- 每个新增题库返回 12 道题；
- 至少抽查一个题库的知识图谱包含节点和关系；
- 首页与健康检查仍返回 HTTP 200。
