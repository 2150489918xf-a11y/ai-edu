# 课程分组删除模式设计

## 目标

移除课程分组后方常驻的独立删除按钮，让日常筛选状态保持整洁，同时保留空分组删除能力。

## 交互

- 在页面顶部操作区增加次要按钮“删除分组”。
- 默认状态下，分组胶囊只显示名称和单元数量，不渲染删除图标，也不为图标预留空间。
- 点击“删除分组”进入删除模式，按钮文案变为“完成删除”。
- 删除模式下，删除图标显示在每个分组胶囊内部，紧跟单元数量。
- 非空分组的删除图标禁用，提示“请先删除分组下的备课单元”。
- 空分组的删除图标可点击，继续使用现有确认弹窗和后端删除接口。
- 点击“完成删除”退出删除模式，所有图标立即隐藏。
- “全部课程”不是数据库课程分组，不显示删除图标。

## 样式

- 删除模式开关复用“新建课程分组”的浅色按钮风格。
- 内嵌删除图标使用红色弱提示，不增加独立胶囊或外边框。
- 图标可视尺寸较小，但使用透明内边距维持足够点击区域。
- 分组筛选条继续使用自身横向滚动，不增加页面级横向滚动。

## 验证

- 页面合同测试确认存在 `groupDeleteMode` 状态和顶部切换按钮。
- 页面合同测试确认删除图标由 `v-if="groupDeleteMode"` 控制并位于分组按钮内部。
- 浏览器检查默认模式没有可见删除图标，切换后图标出现且非空分组禁用。
- 运行生产构建。
