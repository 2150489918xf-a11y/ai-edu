# 题库知识路径图美化设计

## 1. 背景

当前题库知识图谱已经完成题库级动态数据、增量解析、人工编辑和 G6 渲染，但默认画布使用单中心 `radial` 布局。实际数据中节点数量不大但关系密度较高，例如 14 个节点对应 85 条关系，导致节点集中在画布中心、连线大量交叉，知识点标签彼此遮挡，画布空间利用率也很低。

本次改造将页面定位从“关系网络浏览器”调整为“知识路径图”：教师首先看到从基础知识到综合应用的学习路径，复杂关系仍然保留在数据库和节点详情中，但不再全部堆叠在主画布上。

## 2. 目标与范围

### 目标

- 只保留一个主视图：从基础知识到综合应用的知识路径分层图。
- 使用 G6 内置 `antv-dagre` 层次布局，方向为从左到右。
- 根据现有关系类型自动形成层级，不新增 AI 坐标或颜色生成逻辑。
- 老师拖动节点后保存位置，刷新和再次进入页面后保持。
- 首次加载和重新布局时充分利用画布空间，消除当前图谱缩在中心的问题。
- 选中节点时突出上下游路径，弱化无关节点和关系。
- 保留所有关系数据，非路径关系仅在节点详情和关系详情中查看。
- 保持现有题库图谱接口和数据库结构兼容。

### 非目标

- 不新增跨题库总图谱。
- 不改变题目解析、知识点合并、关系编辑和增量同步规则。
- 不让 AI 生成节点坐标、颜色、尺寸或前端布局参数。
- 不在本次改造中新增完整关系网络切换视图。
- 不删除数据库中的 `related`、`co_occurrence` 或 `confusable` 关系。

## 3. 现有实现结论

- 页面入口是 `/question-banks/:bankId/knowledge-graph`。
- `QuestionBankKnowledgeGraphPage.vue` 负责查询、筛选、节点选择、编辑、轮询和布局保存。
- `KnowledgeGraphRenderer.vue` 使用 `@antv/g6` 5.1.1 的 `Graph`。
- 当前布局为 `radial`，使用固定 `unitRadius`，渲染完成后调用 `fitCenter`，没有对图谱边界进行自适应缩放。
- 当前页面把所有关系投影给 G6，主画布没有按关系价值降低数据密度。
- 当前节点布局数据已经存储在 `KnowledgeGraphNodeLayout`，字段包含 `x`、`y` 和 `pinned`，可以支撑本次持久化需求。
- 后端图谱接口已经返回 `category`、`source`、`questionCount`、`orphan`、关系 `type`、`supportCount` 和布局位置等字段。

## 4. 方案选择

采用 G6 内置 `antv-dagre` 层次布局，不采用固定四列泳道，也不采用单根思维导图。

原因：题库知识点具有多前置、多应用、多对多关系，Dagre 能表达有向多分支路径，同时不要求每个知识点只有一个父节点；使用现有 G6 能减少引入新渲染器的风险；关系和布局数据仍由现有后端提供，不需要重新建模。

## 5. 数据投影与路径规则

前端增加一个纯函数投影层，将后端完整图谱转换为路径画布数据。该投影只影响渲染，不修改后端响应。

### 5.1 主路径关系

仅以下关系参与主画布布局和绘制：

| 关系类型 | 画布含义 | 布局作用 |
|---|---|---|
| `prerequisite` | 前置知识 | 将源节点放在目标节点之前 |
| `derivation` | 推导关系 | 将推导起点放在推导结果之前 |
| `application` | 应用于 | 将知识规律指向方法或应用节点 |

关系方向沿数据库中的 `source -> target` 使用。投影层需过滤自环、无效节点和重复关系；关系标签默认不显示，只在选中或悬停时显示。

### 5.2 非路径关系

`related`、`co_occurrence`、`confusable` 不参与主路径布局，也不绘制到主画布。它们继续由后端返回，并在节点详情和关系详情中显示，教师仍可以编辑或隐藏这些关系。

### 5.3 环和孤立节点

- 如果主路径关系形成环，投影层不删除数据库关系，只在布局投影中按置信度、支持题目数量和关系类型做确定性裁剪，保留可形成 DAG 的主路径。
- 没有主路径入边或出边的节点不隐藏，放入画布底部的“待归类知识点”区域，使用低饱和度样式。
- 同一节点拥有多个前置或多个应用节点时全部保留，不强制压缩为单父节点。

## 6. G6 渲染设计

### 6.1 布局

使用 `antv-dagre`：

- `rankdir: 'LR'`，从左到右展示学习路径。
- 使用较大的 `ranksep` 和 `nodesep`，避免层级与节点标签相互遮挡。
- 使用折线边并开启控制点计算，减少长边穿过节点。
- 对自动布局结果执行 `fitView`，而不是只调用 `fitCenter`。
- 首次加载、筛选变化和重新布局时保持当前视图中心，避免不必要的跳动。

### 6.2 节点

- 从圆形改为圆角矩形，名称直接显示在节点内部。
- 名称最多两行，超过后使用省略号，节点宽度根据文本长度限制在稳定范围内。
- 节点底部显示关联题目数量，题目越多节点越醒目，但不使用过大的尺寸差异。
- 主题分类继续使用确定性配色，但改为低饱和背景色与深色文字。
- AI 节点、题目标签节点、人工节点和人工锁定节点通过边框和来源标记区分，不依赖过多颜色。
- 待归类节点使用浅灰背景和虚线边框。

### 6.3 关系

- `prerequisite` 使用深绿色实线箭头。
- `derivation` 使用蓝绿色实线箭头。
- `application` 使用橙色实线箭头。
- 关系标签默认隐藏，选中关系或相关节点时才显示，避免标签堆叠。
- 选中节点时只强化其上下游路径，其他路径降低透明度；不重新创建图实例，避免布局跳变。

### 6.4 画布操作

保留拖拽画布、缩放画布和拖拽节点；增加轻量工具按钮：

- 适应画布。
- 放大。
- 缩小。
- 恢复自动布局。

每个图标按钮提供 `title` 和可访问名称。画布空白区域双击恢复完整路径视图。

## 7. 布局持久化

继续使用 `KnowledgeGraphNodeLayout`，不新增表或字段。

- 自动布局生成未固定节点的位置。
- 节点拖动结束后，使用现有防抖逻辑保存 `x`、`y`、`pinned: true`。
- 进入页面时将已保存的位置作为 G6 的 `preset` 输入；自动布局完成后恢复已固定节点坐标。
- “重新布局”只重新计算未固定节点。
- “恢复自动布局”批量将所有节点设置为 `pinned: false`，随后重新运行 Dagre。
- 数据版本冲突时沿用现有 `graphRevision` 校验，失败后重新加载图谱并提示教师。

## 8. 页面结构与交互

### 页面结构

```text
[返回题库] 题库名称 · 知识路径图                 [新增知识点] [解析未完成题目]

[有效题目] [已解析] [路径节点] [路径关系]       同步状态

[搜索知识点] [来源] [主题]                      [重新布局] [适应画布]

┌──────────────────────────────────────┬──────────────┐
│                                      │ 节点详情     │
│     基础知识 → 核心规律 → 应用方法     │ 关联题目     │
│                                      │ 编辑与关系   │
└──────────────────────────────────────┴──────────────┘
```

主页面移除关系类型筛选和邻域深度筛选，因为主画布已经固定为路径视图。保留搜索、来源和主题筛选；关系类型仍在详情面板中管理。

### 节点详情

右侧详情继续复用 `KnowledgeGraphInspector.vue`，但调整信息顺序：

1. 知识点名称、主题、来源和锁定状态。
2. 前置节点与后续节点。
3. 关联题目。
4. 非路径关系和人工编辑操作。

未选择节点时，详情栏显示简洁空状态；桌面端画布可扩展，移动端详情栏变为抽屉。

## 9. 响应式与视觉规范

- 桌面端保持画布和详情双栏，画布占主要宽度。
- 宽度小于 1180px 时详情栏变为画布下方内容，页面允许自然滚动。
- 宽度小于 760px 时工具栏换行，节点详情使用抽屉或展开区域。
- 页面保留教师端的绿色、米白和深色文字体系，但减少大面积半透明卡片和无意义留白。
- 画布背景使用轻网格，仅作为定位参考，不抢夺节点和路径的视觉权重。
- 所有按钮使用已有 Material Symbols 图标，并为陌生图标提供 tooltip。

## 10. 错误与空状态

- 图谱加载失败：保留页面结构，显示可重试错误条，不显示假数据。
- 题目正在解析：保留已完成路径，顶部显示解析进度。
- 没有知识点：显示题库状态和“解析未完成题目”操作。
- 没有主路径关系：显示待完善路径说明和待归类节点，不使用空白画布。
- Dagre 计算失败：优先使用已持久化坐标；无坐标时使用稳定的网格回退布局。

## 11. 测试与验收

### 单元与静态测试

- 新增路径投影测试：关系类型过滤、自环过滤、重复关系、孤立节点和环裁剪。
- 更新 G6 渲染器测试：使用 `antv-dagre`、`rankdir: 'LR'`、`fitView`、路径关系样式和节点选中弱化。
- 更新页面契约测试：移除关系筛选和邻域筛选，保留搜索、来源、主题、重新布局和详情编辑。
- 保留现有数据库、API、人工编辑和布局持久化测试。

### 浏览器验收

- 使用当前牛顿第二定律题库验证页面首屏不再缩成中心小团。
- 验证路径从左到右可读，节点和标签没有明显遮挡。
- 点击节点后，上下游路径高亮，无关节点弱化，详情信息可点击。
- 拖动节点、刷新页面、重新进入页面，位置保持。
- 恢复自动布局后所有节点重新分层。
- 验证无路径关系、解析中、加载失败和移动端状态。
- 运行 `npm run build` 及知识图谱相关测试。

## 12. 验收标准

- 主画布不再显示密集全关系网络，只展示知识路径。
- 路径关系按左到右层级排列，至少能区分基础、核心和应用三个层级。
- 14 节点、85 关系的现有题库在首次打开时节点可读、连线可追踪、画布空间利用合理。
- 复杂关系没有丢失，可以在详情面板中查询和编辑。
- 节点拖动位置可持久化，自动布局和手动布局之间行为明确。
- 没有新增假数据、假 fallback 或 AI 坐标参数。
