# 题库知识图谱教学路径分层设计

## 背景

当前知识图谱已经改为从左到右的 Dagre 路径图，但仍同时展示大量跨层关系。节点位置虽然紧凑，视觉上仍像算法布局结果：层级边界不明确、长边穿越整个画布、教师难以快速判断学习顺序。

本次采用已确认的方案 A，将主画布改造成四阶段教学路径图。

## 目标

- 明确展示“基础概念 → 核心规律 → 解题方法 → 综合应用”四个教学阶段。
- 节点固定落在对应阶段列中，避免自由布局形成不规则折线网。
- 默认只显示教学主路径，减少跨层长边和视觉噪声。
- 点击节点后仍能查看与该节点有关的跨层关系，不丢失图谱信息。
- 保持现有题库接口、数据库结构、关系编辑和节点详情功能不变。

## 阶段投影

前端新增纯函数教学阶段投影，不修改后端数据。

### 阶段名称

| 索引 | 阶段 |
|---|---|
| 0 | 基础概念 |
| 1 | 核心规律 |
| 2 | 解题方法 |
| 3 | 综合应用 |

### 自动归并规则

1. 使用现有路径投影得到无环关系和拓扑层级 `layer`。
2. 路径层级不超过四层时保留原层级顺序，并将最后一层映射到第四阶段，使学习起点和最终应用保持视觉边界。
3. 路径层级超过四层时，按层级在完整深度中的相对位置线性归并到四个阶段。
4. 同阶段节点按 `category`、`questionCount`、`label` 和 `id` 稳定排序。
5. 孤立节点放入“基础概念”阶段末尾，保留虚线边框并视为待归类节点。

阶段计算必须确定性，同一份数据刷新后位置不随机变化。

## 画布布局

不再使用 Dagre 自动坐标作为主布局，改为固定四列的教学路径坐标：

- 四列拥有稳定的横向中心点和统一列宽。
- 每列节点按固定垂直间距排列，列内整体垂直居中。
- 节点仍使用圆角矩形，保持题目数量、分类颜色、来源和锁定状态。
- 画布背景增加四个低对比度阶段区域和阶段标题。
- 宽屏下四列并排；窄屏仍由 G6 `fitView` 整体缩放，页面布局继续使用现有移动端堆叠规则。

## 关系显示

### 默认状态

- 常显相邻阶段之间的路径关系。
- 同阶段关系和跨越两个及以上阶段的关系不常显，避免长线穿过多个节点列。
- 前置、推导、应用三种关系继续使用现有绿色、蓝绿色和橙色语义。

### 选中状态

- 点击节点后，显示该节点所有保留路径关系，包括同阶段和跨阶段关系。
- 选中节点及其直接上下游保持高对比，其他节点和关系降低透明度。
- 跨阶段关系使用较细的虚线或低透明曲线，与默认主路径区分。
- 右侧检查器继续显示完整路径关系和辅助关系。

## 交互和持久化

- 保留放大、缩小、适应画布和节点点击。
- 固定教学列布局下取消节点自由拖动和坐标保存，避免人工拖动破坏四阶段结构。
- 页面原有“重新布局”改为重新计算四阶段投影。
- “恢复自动布局”不再写入节点坐标，仅恢复四阶段默认顺序。
- 现有布局数据继续保留在数据库中，但教学路径视图不读取其坐标。

## 错误与边界状态

- 无路径关系时仍展示四阶段背景，所有节点进入基础概念列并保留待归类样式。
- 单节点图谱将节点放在基础概念列并适应画布。
- 某个阶段无节点时保留阶段标题和空列，不移动其他阶段。
- 节点数量较多时扩大画布世界坐标高度，再由 `fitView` 适配，防止节点重叠。

## 测试与验收

- 纯函数测试覆盖一至四层、五层以上、孤立节点、稳定排序和阶段边界。
- 渲染器测试检查四阶段背景、固定坐标布局、默认主关系和选中跨层关系。
- 页面测试检查旧布局保存入口不再触发坐标持久化。
- 浏览器使用当前牛顿定律题库验证：
  - 四阶段边界清晰；
  - 默认画面没有贯穿全图的长边；
  - 点击节点可恢复该节点跨层关系；
  - 节点与标签无明显遮挡；
  - 桌面、平板和手机宽度下页面不横向溢出。
- 运行相关知识图谱测试和 `npm run build`。
