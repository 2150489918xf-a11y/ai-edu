# 题库知识路径图紧凑布局设计

## 背景

当前知识路径图已经使用 G6 `antv-dagre` 从左到右分层，但默认层间距、同层间距和画布适配留白偏大。跨层关系进一步扩大了布局边界，导致节点在首屏中显得松散、文字偏小。

## 目标

- 在不改变图谱数据、关系方向和人工编辑能力的前提下，提高节点密度与首屏可读性。
- 保持路径从左到右、节点不重叠、连线可追踪。
- 保留节点拖动、固定位置、缩放和适应画布能力。

## 方案

继续使用 `antv-dagre`，仅调整渲染参数：

- `ranksep` 从 `150` 调整为 `88`，缩短相邻层的水平距离。
- `nodesep` 从 `64` 调整为 `28`，缩短同层节点的垂直距离。
- `fitView` 的 `padding` 从 `56` 调整为 `28`，减少画布四周无效留白。
- 节点宽度限制从 `128-190` 调整为 `116-172`，维持稳定尺寸并减少长文本对布局边界的拉伸。
- 折线边继续使用 Dagre 控制点；将圆角半径适度降低，避免长边转折占用过多空间。

## 不变范围

- 不改变路径投影、环裁剪、孤立节点或关系过滤规则。
- 不清除已保存的人工固定坐标。
- 不新增布局模式、数据库字段或后端接口。
- 不让 AI 生成坐标、间距或前端样式参数。

## 响应式行为

- 桌面端使用紧凑参数完成首次适配。
- 窄屏仍由 G6 `fitView` 自动缩放，用户可以使用现有放大、缩小和适应画布按钮。
- 不改变页面双栏和移动端堆叠规则。

## 验收标准

- 相同 14 个节点和 12 条路径关系的首屏占用范围明显收紧。
- 节点文字仍可辨识，节点之间无明显重叠。
- 长边不穿过节点，方向箭头和关系颜色保持可识别。
- 节点点击、拖动和轮询期间画布稳定性不回归。
- 渲染器契约测试和生产构建通过。
